5 # MCH Raw Data Decoder
6 
7 There are currently two data formats to be dealt with : the "bare" one coming
8 out of the CRU when no user logic (UL) is used, or the "UL" one where user
9 logic is actually used. Each of the two formats can be used in charge sum (the
10 sampa electronics does the charge integration) or in sample mode (the sampa
11 electronics transmit all the charge samples). All formats are composed of
12 {RawDataHeader,Payload} pairs.
13 
14 Note that the cluster sum mode has to be properly selected, as there is
15 unfortunately no way to infer it from the data itself...
16 
17 ## createDecoder&lt;FORMAT,CHARGESUM,RDH>
18 
19 On the reading/consumer/decoding end, the choice of the decoder to use is made
20 through a templatized function, `createDecoder<FORMAT,CHARGESUM,RDH>`.
21 Currently the following template parameters combinations have been tested :
22 
23 | FORMAT | CHARGESUM | RDH |
24 | :-------------: | :-----------: | :-------------: |
25 | BareFormat | ChargeSumMode | RawDataHeaderV4 |
26 | BareFormat | SampleMode | RawDataHeaderV4 |
27 | UserLogicFormat | ChargeSumMode | RawDataHeaderV4 |
28 | UserLogicFormat | SampleMode | RawDataHeaderV4 |
29 
30 RDH V5 is not yet supported, but is planned.
31 
32 As an example, to get a decoder for BareFormat in ChargeSum mode, using
33 RawDataHeaderV4 :
34 
35 ```.cpp
36 #include "MCHRawDecoder/Decoder.h"
37 
38 RawDataHeaderHandler<RAWDataHeaderV4> rh;
39 SampaChannelHandler ch;
40 
41 auto decoder = o2::mch::raw::createDecoder<BareFormat, ChargeSumMode, RAWDataHeaderV4>(rh, ch);
42 
43 // get some memory buffer from somewhere ...
44 buffer = ...
45 
46 // decode that buffer
47 decode(buffer);
48 ```
49 
50 The `createDecoder` function requires two parameters : a `RawDataHeaderHandler`
51 and a `SampaChannelHandler`.
52 Both parameters can be defined as lambdas, regular free functions, or member
53 functions of some class.
54 
55 ## RawDataHeaderHandler
56 
57 The `RawDataHeaderHandler` is a function that takes a RawDataHeader and
58 (optionally) returns a RawDataHeader, i.e. :
59 
60 ```.cpp
61 using RawDataHeaderHandler = std::function<std::optional<RDH>(const RDH& rdh)>;
62 ```
63 
64 If no RDH is returned (i.e. `std::nullopt` is returned) then that part of the
65 raw data is *not* decoded at all.
66 The returned RDH can be a modified version of the argument, e.g. to change some
67 of the values (like `feeId` according to some mapping).
68 
69 ## SampaChannelHandler
70 
71 The `SampaChannelHandler` is also a function, that takes a dual sampa
72 identifier (in electronics realm, aka solar,group,index tuple), a channel
73 identifier within that dual sampa, a `SampaCluster` and returns nothing, i.e. :
74 
75 > A word of caution here about the naming. A `SampaCluster` is a group of raw
76 > data samples of *one* dual sampa channel, and has nothing to do with a
77 > cluster of pads or something alike.
78 
79 ```.cpp
80 using SampaChannelHandler = std::function<void(DsElecId dsId,
81  uint8_t channel,
82  SampaCluster)>;
83 ```
84 
85 That function is called by the raw data decoder for each `SampaCluster` that it
86 finds in the data.
87 A very simple example would be a function to dump the SampaClusters :
88 
89 ```.cpp
90 SampaChannelHandler handlePacket(DsElecId dsId, uint8_t channel, SampaCluster sc) {
91  std::cout << fmt::format("{}-ch-{}-ts-{}-q-{}", asString(dsId), channel, sc.timestamp, sc.chargeSum));
92 }
93 // (note that this particular function only correctly handles the SampaCluster in ChargeSum Mode)
94 ```
