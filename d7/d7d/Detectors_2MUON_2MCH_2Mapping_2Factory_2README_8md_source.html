5 A single utility function to create segmentations, avoiding duplications
6 of objects.
7 
8 For instance :
9 
10 ```cpp
11 const Segmentation& seg1 = segmentation(501);
12 ...
13 (some code here)
14 ...
15 auto& seg2 = segmentation(501);
16 ```
17 
18 will result in only one object for segmentation 501 to be created.
19 
20 On the contrary, if you use :
21 
22 ```cpp
23 Segmentation seg1{501};
24 ...
25 ...
26 Segmentation seg2{501};
27 ```
28 
29 two separate objects will be created (and you will be hit twice by the
30 object creation time).
31 
32 Note that depending on your use case, both approaches might be valid ones.
33 While the factory one ensures you get only get one object for each detection
34 element, it creates all detection elements at once. So, if you only need the
35 segmentation of one detection element, you'd better off *not* using the
36 factory.
